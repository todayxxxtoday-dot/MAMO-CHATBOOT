# دليل نشر المتجر على Firebase Hosting

يمكنك نشر هذا المشروع الفول ستاك (React + Express + Firebase) على خوادم **Firebase Hosting** ليكون متاحاً للجمهور بسهولة وبخطوات بسيطة.

---

## 📋 المتطلبات الأساسية
قبل البدء، تأكد من تثبيت أدوات Firebase CLI على جهازك المحلي عبر ترقية NPM:
```bash
npm install -g firebase-tools
```

---

## ⚡ خطوات تهيئة ونشر المشروع

### ١. تسجيل الدخول إلى حساب Firebase
افتح ملخص سطر الأوامر في مجلد المشروع وسجل الدخول باستخدام حساب جوجل الخاص بك:
```bash
firebase login
```

### ٢. تهيئة المشروع (Initialization)
ابدأ عملية إعداد بيئة الاستضافة داخل المشروع عبر استدعاء الأمر التالي:
```bash
firebase init
```

أثناء عملية الإعداد التفاعلية، تفضل باختيار التالي:
1. واختر خيار **Firestore** و **Hosting**:
   - `Firestore: Configure security rules`
   - `Hosting: Configure files for Firebase Hosting`
2. اختر المشروع المخصص الخاص بك من لوحة المشاريع المتوفرة (**Use an existing project**).
3. عند سؤالك عن مجلد الملفات العامة (**What do you want to use as your public directory?**)، اكتب **`dist`** (وهو المجلد الذي يتم إنتاج ملفات بناء React الثابتة بداخله).
4. عند سؤالك عما إذا كان التطبيق صفحة واحدة (**Configure as a single-page app?**)، اختر **`Yes`** (نعم).
5. عند سؤالك حول البناء التلقائي بـ GitHub، اختر **`No`** (لا).

---

## 🚀 البناء والنشر الفعلي (Deploy)

عند الرغبة في إطلاق التطبيق وحقن وحفظ البيانات الحية:

### ١. بناء كود الواجهة الأمامية والمخدم
قم بإتاحة عملية البناء والترجمة لمشروعك عبر استدعاء:
```bash
npm run build
```

### ٢. نشر القواعد وقواعد الاستضافة
يمكنك رفع قواعد البيانات وقوانين الأمان وقرار الاستضافة إلى السحابة بأمر واحد:
```bash
firebase deploy
```

بعد اكتمال عملية الرفع، سيوفر لك سطر الأوامر رابط استضافة مباشر محمي ببروتوكول HTTPS (مثل `https://your-app.web.app`) لتبادل التطبيق مع عملائك وتجربته فوراً وبسلاسة تامة.
